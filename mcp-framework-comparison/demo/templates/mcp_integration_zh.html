MCP 框架集成指南

比较 LlamaIndex、LangChain、SmolaGents 和 AutoGen 与 MCP 的集成

English

什么是 MCP？

模型上下文协议（Model Context Protocol，MCP）是一个标准化协议，用于在 AI 应用程序中集成外部工具和资源。它提供了：

  • 统一的工具调用接口
  • 标准化的资源访问方式
  • 跨框架的互操作性
  • 可扩展的插件系统

框架集成比较

框架 集成方式 代码复杂度 性能 灵活性 错误处理
LlamaIndex 自定义检索器 中等 优秀 高 内置
LangChain 工具包装器 低 良好 高 工具级
SmolaGents 函数调用 低 良好 中等 基础
AutoGen 多代理 中高 优秀 非常高 全面

核心集成代码

class MCPRetriever(BaseRetriever):
    def __init__(self, client: MCPClient):
        self.client = client
        super().__init__()
    
    def _retrieve(self, query_bundle: QueryBundle) -> List[NodeWithScore]:
        query = query_bundle.query_str
        results = self.client.call_tool(
            "knowledge_base_search",
            {"query": query}
        )
        nodes = []
        for result in results:
            node = TextNode(text=result["content"])
            nodes.append(NodeWithScore(node=node, score=result["score"]))
        return nodes

# 使用示例
retriever = MCPRetriever(client)
query_engine = RetrieverQueryEngine(retriever)
response = query_engine.query("查询示例")

def create_mcp_knowledge_tool(client: MCPClient) -> Tool:
    def knowledge_tool_fn(query: str) -> str:
        result = client.call_tool(
            "knowledge_base_search",
            {"query": query}

核心集成代码

class Tool:
    def __init__(self, name: str, description: str, function: Callable):
        self.name = name
        self.description = description
        self.function = function
    
    def __call__(self, *args, **kwargs):
        return self.function(*args, **kwargs)

def create_mcp_tool(client: MCPClient) -> Tool:
    def tool_fn(query: str) -> str:
        result = client.call_tool(
            "knowledge_base_search",
            {"query": query}
        )
        return json.dumps(result, indent=2)
    
    return Tool(
        name="mcp_tool",
        description="MCP 工具",
        function=tool_fn
    )

# 使用示例
agent = Agent(tools=[create_mcp_tool(client)])
response = agent.run("查询示例")

最佳实践

  • 根据项目需求选择合适的框架：
    • 数据检索和文档处理 → LlamaIndex
    • 工具链和代理系统 → LangChain
    • 轻量级应用 → SmolaGents
    • 复杂多代理系统 → AutoGen
  • 注意错误处理和异常管理
  • 实现适当的日志记录和监控
  • 考虑性能优化和缓存策略
  • 保持代码模块化和可维护性
